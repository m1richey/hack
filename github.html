Tutorials

Using GitHub

GitHub is used to help project teams easily share files. This tutorial describes how to install git and set up a GitHub account.

Signing Up

  1. Navigate to github.com.
  2. If you already have an account, choose ‘Sign In’ and enter your username and password.
  3. If you do not have an account, choose a username and password and enter your email on the homescreen. Then, click ‘Sign Up for GitHub’.

You have now created an account! You will be brought to the homepage, from which you can access tutorials as well as repositories you have created. An outline of these tutorials is provided below.

Creating a Repository

A repository is a location to store shared code. You and your teammates will use this to coordinate each piece of code you edit.

  1. On the right side of the page, choose ‘New Repository’.
  2. Enter a name for the repository. Names should be short and easy to remember.
  3. Choose who can have access to your repository. Choose public if any GitHub user can view it. Choose private if only teammates you choose can view it.
  4. Check the box ‘Initialize this repository with a README. Ensure the dropdown menus are set to ‘None’.
  5. Choose ‘Create Repository’.

You have now created a new repository! In the center of the screen, you can view all files stored in the repository.

Creating a New File to Store in the Repository

Once you have a repository set up, you need a new file to store and edit.

  1. Navigate to the repository page you just created. At the top of the page, it should display /
  2. Choose the ‘+’ symbol next to the repository name in the middle of the page.
  3. At the top of the page, name your file. Add the appropriate extension for each type of file (i.e. for a C language file).
  4. Enter the content of your file in the black box in the center of the screen.
  5. Once you are done adding content, scroll to the bottom of the page and choose “Commit New File”.
  6. Your file will now be displayed in the list of files under the repository.

Saving the Repository to your Personal Computer

The above tutorials have explained how to add files and edit the repository through the GitHub website, but now we will cover how to save the repository to your personal computer so that you can add pre-existing files.

  1. Ensure that git is installed on your computer. This tutorial will not cover how to install git, but user manuals are provided at the following website: Installing Git
  2. Ensure that a command shell and ssh are installed on your computer. This tutorial will not cover how to install these tools, but user manuals are provided at the following website: Installing Command Line
  3. Go to GitHub.com and navigate to your repository page.
  4. On the right side of the page, locate the “Clone URL” dialogue box.
  5. Under the box, choose the “SSH” option.
  6. Copy the URL that appears in the dialogue box to your clipboard.
  7. Open your command shell and navigate to the folder in which you want to store the repository.
  8. Type ‘git clone ’
  9. The screen will say “Cloning into ”. This process will take approximately 30 seconds.
  10. Ensure that the entire repository has been cloned by navigating into the folder. Check that all files that appear on the GitHub.com repository page are present in the folder.

The repository has been copied to your computer! You are now able to edit and add new content to the repository from your own machine.

Adding an Existing File to the Repository

Now that the repository exists on your personal computer, you can add existing files to the repository webpage.

  1. Copy the file you want to add to the repository into the folder in which you cloned the repository in the steps above.
  2. Type ‘git add ’
  3. Type ‘git commit -m “”
  4. Ensure that the file has been added by navigating to the repository page on GitHub.com and checking that the file listings have uploaded.

The file is now added to the repository, and other repository users can can access your file.
